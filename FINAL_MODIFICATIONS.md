### 修改 3️⃣：五個監控區塊持續更新 + 模擬觸發訊息

**檔案**: `app.py`

#### 修改內容：
- ✅ 新增：模擬資料生成函數
- ✅ 新增：模擬股票資料池（15支常見股票）
- ✅ 新增：每 10 秒自動生成一筆模擬觸發訊息
- ✅ 新增：隨機分配到五個監控區塊中的任一個
- ✅ 新增：觸發時自動保存狀態（persist/daily_state.json）

#### 實現方式：

**模擬股票池**：
```python
_MOCK_STOCKS = [
    {"symbol": "2330", "name": "台積電", "basePrice": 550},
    {"symbol": "2454", "name": "聯發科", "basePrice": 850},
    # ... 共 15 支常見股票
]
```

**模擬觸發生成**：
```python
def _generate_mock_trigger():
    """每 10 秒生成一筆新觸發"""
    stock = random.choice(_MOCK_STOCKS)
    return {
        "symbol": stock["symbol"],
        "name": stock["name"],
        "price": 隨機價格,
        "changePercent": 隨機漲跌幅,
        "triggerTime": 觸發時間,
    }

def _add_mock_triggers():
    """隨機添加到五個監控清單之一"""
    choice = random.randint(0, 4)
    if choice == 0:  # 開盤出量
        state.triggered_opening_volume.insert(0, trigger_data)
    elif choice == 1:  # 漲跌停打開
        state.triggered_limit_break.insert(0, trigger_data)
    # ... 其他監控區塊
```

**頁面重整時調用**：
```python
# 每次重整前生成模擬訊息
if config.DEMO_MODE:
    _add_mock_triggers()

time.sleep(5)
st.rerun()
```

#### 視覺效果：
- 五個監控區塊會不定期出現新的觸發訊息
- 每筆觸發訊息包含完整資訊（代號、名稱、價格、漲跌幅、時間等）
- 新觸發訊息自動插入清單最上方（最新優先）
- 每筆觸發最多保留 20 筆，防止清單過長
- 觸發訊息會自動保存，重開 app 時保留
